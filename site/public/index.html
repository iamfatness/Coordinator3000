AI coordination engine

Label an issue. Get a reviewed pull request.

Coordinator3000 is a minimal-HITL, autonomous multi-agent system. Tag a GitHub issue ai-task and it clones the repo, plans the work, writes the code, reviews it, and opens a PR — no one babysitting the run.

What it does

🏷️

Issue-triggered

A GitHub webhook fires the moment an issue is labeled ai-task. No CLI, no manual kickoff.

🧭

Orchestrated agents

An Orchestrator routes between a Coder and a Reviewer in a LangGraph state machine, iterating until the change is solid.

🔀

Auto pull requests

Auto-creates a branch, commits, pushes, comments on the issue, and opens a PR. Ready-for-review when the automated review passes.

🤖

Multi-model

Each agent can run on Claude Grok or Codex — set per role, no code changes.

📦

Sandboxed execution

Agents build and run tests in a workspace-scoped sandbox (subprocess or hardened Docker) with a path-containment guard.

💾

Durable runs

Every run is checkpointed in Postgres, so it survives restarts and is inspectable and resumable by thread id.

How a run flows

Label the issue

Add ai-task. The webhook is HMAC-verified and enqueued to a background worker.

Clone & branch

The worker clones the repo and cuts a fresh ai-task/issue-N-… branch.

Plan

The Orchestrator analyzes the issue and routes the first step to the Coder.

Code

The Coder edits files, runs the test suite in the sandbox, and commits.

Review

The Reviewer inspects the diff, re-runs tests, and returns a pass/fail verdict — looping back if needed.

Open PR

Finalize pushes the branch, opens the pull request, and comments the result on the issue.

The console

See every run at a glance.

A live dashboard shows each run's pipeline progress, the model behind each agent, iteration count, and the resulting PR — with a detail drawer for the plan, timeline, diff, and review. Trigger runs by hand for any owner/repo/issue, no label required.

Quickstart

Bring up Postgres + the app, point a GitHub webhook at it, and label an issue.

# 1. configure secrets
cp .env.example .env   # GITHUB_TOKEN, GITHUB_WEBHOOK_SECRET, ANTHROPIC_API_KEY

# 2. run app + postgres
docker compose up --build

# 3. point a GitHub "Issues" webhook at:
POST https://your-host/webhooks/github

# 4. label any issue `ai-task` — or trigger from the console:
POST /api/runs  {"owner":"you","repo":"repo","issue_number":7}

Full setup, configuration, and security notes are in the README.
